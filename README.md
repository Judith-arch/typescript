## Module 1 — Primitive Types, Inference and Pure Logic

### Concepts Covered

- Primitive types: `string`, `number`, `boolean`
- Type inference and `as const`
- Special types: `void`, `unknown`, `never`
- Strictly typed arrays and tuples
- Function signatures with `number | null` return for edge cases
- Narrowing with `typeof`

### Implemented Functions (`math-utils.ts`)

| Function | Description | Return type |
|---|---|---|
| `calcularMedia(datos)` | Arithmetic mean | `number \| null` |
| `calcularMediana(datos)` | Median (without mutating the array) | `number \| null` |
| `filtrarAtipicos(datos, limite)` | Removes values far from the mean | `number[]` |
| `calcularDesviacionEstandar(datos)` | Sample standard deviation | `number \| null` |

All functions return `null` for empty arrays — an explicit edge case required by `strictNullChecks`.

### Running the Module

```bash
cd module-1
npm install
npm run dev      # runs with tsx
npm run build    # compiles to dist/
npm run check    # type check without compiling (0 errors)
```

## Module 2 — Complex Data Modelling, Patterns and Generics

### Concepts Covered

- `interface` vs `type`: design criteria
- `readonly` and optional properties
- Union (`|`) and intersection (`&`) types
- **Discriminated unions** with a shared literal discriminant property
- Generic programming `<T>` with `extends` constraints
- `Record<K, T>` for typed dictionaries
- Strongly typed promises with `async/await`

### Domain Model (`university.types.ts`)

```
Estudiante          — entity with readonly id and optional last access
Asignatura          — entity with credits and mandatory/optional flag
EstudianteAuditable — intersection: Estudiante & EntidadAuditable
EstadoMatricula     — discriminated union of three states:
  ├── MatriculaActiva      (tipo: "ACTIVA"      → asignaturas[])
  ├── MatriculaSuspendida  (tipo: "SUSPENDIDA"  → motivoSuspension)
  └── MatriculaFinalizada  (tipo: "FINALIZADA"  → notaMedia)
RespuestaAPI<T>     — generic wrapper for network responses
```

### Generic Service (`api-client.ts`)

```ts
apiClient.obtenerRecurso<Estudiante>("/estudiantes")   // → Promise<RespuestaAPI<Estudiante[]>>
apiClient.obtenerPorId<Estudiante>("/estudiantes", id) // → Promise<RespuestaAPI<Estudiante | null>>
```

Simulates network latency with `setTimeout` and handles 404 errors with strict typing.

### Running the Module

```bash
cd module-2
npm install
npm run dev      # runs with tsx — shows reports and API calls
npm run check    # 0 TypeScript errors
```

### Architecture Documentation

See [`modulo-2/docs/arquitectura.md`](./module-2/docs/arquitecture.md) for a detailed explanation of design decisions: why `interface` over `type`, the advantage of discriminated unions over ambiguous optional properties, and how generics eliminate duplication without sacrificing type safety.
